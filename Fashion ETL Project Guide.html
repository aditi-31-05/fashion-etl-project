You can use SQLPlus (Oracle) instead of PostgreSQL—but small reality check: SQLPlus is just a client tool, not the database itself. You’ll be working with an Oracle Database, and SQL*Plus is how you interact with it.

That said, your ETL project will work perfectly fine with Oracle 👍 — just a few changes.

🔷 How your project changes (PostgreSQL → Oracle)

✅ What stays SAME

  • Python (Pandas)

  • CSV dataset

  • Data cleaning

  • ETL concept

🔁 What changes

  • Database = Oracle

  • Connection method = cx_Oracle

  • Tool = SQL*Plus

🔷 Step-by-step setup for your Fashion ETL (Oracle)

1️⃣ Install required things

You’ll need:

  • Oracle Database (or Oracle XE)

  • SQL*Plus

  • Python library:

Bash
pip install cx_Oracle

2️⃣ Connect Python to Oracle

Example:

Python
import cx_Oracle

dsn = cx_Oracle.makedsn("localhost", 1521, service_name="XE")

conn = cx_Oracle.connect(
    user="your_username",
    password="your_password",
    dsn=dsn
)

cursor = conn.cursor()

3️⃣ Create Table (in SQL*Plus)

Open SQL*Plus and run:

SQL
CREATE TABLE fashion_data (
    id NUMBER,
    product_name VARCHAR2(255),
    brand VARCHAR2(100),
    category VARCHAR2(100),
    price NUMBER,
    rating NUMBER
);

4️⃣ Load Data from Python

You CANNOT use .to_sql() directly like PostgreSQL.
Instead use insert loop:

Python
for index, row in df.iterrows():
    cursor.execute("""
        INSERT INTO fashion_data (id, product_name, brand, category, price, rating)
        VALUES (:1, :2, :3, :4, :5, :6)
    """, (
        int(row['id']),
        row['product_name'],
        row['brand'],
        row['category'],
        int(row['price']),
        float(row['rating'])
    ))

conn.commit()

5️⃣ Run SQL Queries (in SQL*Plus)

Example:

SQL
SELECT brand, AVG(price)
FROM fashion_data
GROUP BY brand
ORDER BY AVG(price) DESC
FETCH FIRST 5 ROWS ONLY;
